# 역할 기반 알림 채널 참여와 공지 설계

## 목표

새 서버 구성원에게는 처음에 최상위 `공지`, `안내` 채널만 보여준다. 구성원이 안내 메시지의 버튼을 누르면 `영화 알림` 역할을 부여하고 영화 예매 알림 카테고리를 열어준다. 일반 사용자는 모든 서비스 채널에서 메시지와 스레드를 작성할 수 없으며, 봇과 Discord 관리자만 메시지를 작성할 수 있다.

## 채널 구조

`/알림 초기화`는 다음 구조를 생성하거나 기존 구조를 갱신한다.

```text
공지
안내
영화 예매 알림
├─ 제어
├─ 서버-상태
├─ 메가박스-코엑스-돌비
├─ 메가박스-남현아-돌비
├─ cgv-용산-imax
├─ cgv-용산-4dx
└─ cgv-용산-screenx
```

`공지`와 `안내`는 카테고리에 포함하지 않는 최상위 텍스트 채널이다. 서버에 가입한 모든 구성원이 바로 볼 수 있다. `영화 예매 알림` 카테고리는 `영화 알림` 역할을 받은 구성원에게만 표시한다. `제어` 채널은 역할과 관계없이 초기화한 소유자와 봇만 접근한다.

이 권한은 서비스가 생성한 채널에만 적용하며 서버의 다른 채널에는 영향을 주지 않는다. Discord `Administrator` 권한을 가진 구성원은 플랫폼 규칙에 따라 채널 제한을 우회할 수 있다.

## 영화 알림 역할

초기화할 때 `영화 알림` 역할을 한 개 생성하고 ID를 저장한다. 저장된 역할이 유효하면 재사용하고, 삭제됐으면 같은 목적의 역할을 다시 생성한다. 역할에는 서버 관리 권한을 부여하지 않는다. 채널별 권한 덮어쓰기를 통해 알림 카테고리를 보고 기록을 읽고 이모지 반응을 추가할 수 있게 한다.

봇 역할은 `영화 알림` 역할보다 위에 있어야 하며 길드 수준의 `Manage Roles`, `Manage Channels` 권한을 가져야 한다. 봇이 만든 일반 역할은 봇 역할 아래에 위치시키고 사용자에게 해당 역할만 추가한다.

## 안내 채널과 참여 버튼

`안내` 채널에는 서비스 설명과 `알림 채널 보기` 버튼이 포함된 일반 Markdown 메시지 하나를 생성한다. Embed는 사용하지 않는다. 안내 메시지는 다음 정보를 포함한다.

- 서비스가 영화 예매 오픈을 감시하고 Discord로 알린다는 설명
- 지원하는 메가박스 및 CGV 영화관과 특별관
- `서버-상태`와 영화관별 채널의 용도
- 알림의 앱 열기 버튼 사용 방법
- `알림 채널 보기` 버튼을 누르면 관련 채널이 표시된다는 설명

버튼의 custom ID는 고정하고, 초기화를 다시 실행하면 기존 안내 메시지를 갱신한다. 버튼을 누른 사용자가 이미 `영화 알림` 역할을 가지고 있으면 아무것도 변경하지 않는다. 역할이 없으면 봇이 역할을 추가한다. 성공 응답은 `알림 채널을 열었습니다`이며 버튼을 누른 사용자에게만 보이는 ephemeral 메시지로 전송한다.

역할 부여는 단방향이다. 역할 제거 버튼과 탈퇴 명령은 제공하지 않는다. 역할이 삭제됐거나 봇보다 위에 있어 부여할 수 없으면 사용자에게 간결한 실패 메시지를 보내고 상세 오류는 서버 로그에 남긴다. 역할 부여에는 가입 이벤트나 `Guild Members Intent`를 사용하지 않는다.

## 공지 채널과 소유자 명령

`공지` 채널은 기능 업데이트, 지원 대상 변경, 예정된 점검, 조회 주기 변경, 장기간 서비스 중단 및 재개처럼 구성원이 계속 알아야 하는 내용을 게시한다. 일시적인 공급자 조회 오류와 복구는 기존 `서버-상태` 채널에만 게시하고, 앱 재시작은 공지하지 않는다.

`/알림 공지 내용:<메시지>` 명령을 추가한다. 설치 소유자만 실행할 수 있으며 다른 사용자의 실행은 ephemeral 권한 오류로 거부한다. 아직 초기화되지 않았으면 먼저 초기화하라는 오류를 반환한다. 내용은 앞뒤 공백을 제거한 뒤 빈 문자열을 거부하고 Discord 일반 메시지 제한 안에서 전송한다.

봇은 다음 형태의 일반 Markdown 메시지를 `공지` 채널에 게시한다.

```text
📢 **공지**
<소유자가 입력한 내용>
```

입력에 `@everyone`, `@here`, 사용자 또는 역할 멘션 문법이 있어도 실제 멘션을 발생시키지 않는다. 명령 성공 여부는 실행자에게 ephemeral 메시지로 응답한다.

최초 `/알림 초기화`는 소유자가 없으므로 처음 실행한 사용자를 소유자로 지정한다. 이후 `/알림 초기화`, `/알림 공지`, 제어 패널 선택 및 ON/OFF 기능은 기존처럼 소유자만 실행할 수 있다.

## 권한

`공지`와 `안내`에서 `@everyone`은 채널 보기, 메시지 기록 읽기, 반응 추가만 할 수 있다. 메시지 보내기, 공개·비공개 스레드 만들기, 스레드에 메시지 보내기는 거부한다. 봇은 채널 보기, 기록 읽기, 메시지 보내기와 권한 관리에 필요한 권한을 가진다.

`영화 예매 알림` 카테고리와 `서버-상태`, 영화관별 채널에서 `@everyone`은 채널 보기가 거부된다. `영화 알림` 역할은 채널 보기, 기록 읽기, 반응 추가만 허용되고 메시지 및 스레드 작성은 거부된다. 봇은 메시지를 게시하고 권한 덮어쓰기를 관리하는 데 필요한 권한을 가진다.

`제어` 채널은 `@everyone`의 보기를 거부한다. 소유자는 채널 보기, 기록 읽기와 제어 구성요소 사용만 가능하고 일반 메시지는 보낼 수 없다. 봇은 보기, 기록 읽기와 메시지 보내기가 가능하다. 일반 사용자는 `영화 알림` 역할을 가지고 있어도 `제어` 채널을 볼 수 없다.

## 데이터 저장과 재초기화

`installations`에 다음 ID를 추가한다.

- `viewer_role_id`: `영화 알림` 역할
- `notice_channel_id`: 최상위 `공지` 채널
- `guide_channel_id`: 최상위 `안내` 채널
- `guide_message_id`: 안내 및 참여 버튼 메시지

기존 설치는 마이그레이션에서 빈 값으로 시작한다. 다음 `/알림 초기화`에서 역할과 채널을 생성하고 기존 알림 카테고리 권한을 역할 기반으로 전환한다. 현재 서버에는 기존 일반 구성원이 없으므로 기존 구성원에게 역할을 일괄 부여하지 않는다.

Discord 리소스가 생성된 뒤 후속 단계가 실패해도 ID가 유실되지 않도록 역할, 최상위 채널, 카테고리 같은 주요 리소스 생성 단계마다 설치 정보를 저장한다. 저장된 ID가 삭제된 리소스를 가리키면 동일한 이름과 타입의 유효한 리소스를 우선 재사용하고, 없으면 다시 생성한다. 초기화를 반복해도 역할, 채널 및 안내 메시지가 중복 생성되지 않아야 한다.

권한 전환은 `공지`와 `안내`, 참여 역할을 먼저 준비한 다음 알림 카테고리를 숨긴다. 전환 도중 실패하더라도 일반 사용자가 접근 방법 없이 알림 카테고리만 잃는 상태를 만들지 않는다. 기존 `제어` 채널은 공개되지 않도록 비공개 권한을 먼저 보장한다.

## 오류 처리

- 역할·채널·안내 메시지 생성 또는 갱신 실패는 `/알림 초기화` 실패 응답에 단계 이름과 함께 표시한다.
- 역할 부여 실패는 버튼 사용자에게 간결하게 알리고 상세 Discord 오류는 서버 로그에 기록한다.
- 공지 전송 실패는 실행자에게 간결하게 알리고 공지 채널에는 불완전한 메시지를 남기지 않는다.
- 안내 및 공지 기능의 실패는 영화관 조회와 기존 예매 알림 데이터에 영향을 주지 않는다.
- 채널이 삭제된 경우 기존 영화관 채널 복구 정책을 유지한다.

## 검증

- 신규 초기화가 `영화 알림` 역할, 최상위 `공지`·`안내`, 역할 제한 카테고리와 기존 하위 채널을 생성하는지 검증한다.
- 일반 사용자가 처음에는 `공지`·`안내`만 보고 서비스 채널에서는 메시지와 스레드를 작성할 수 없는 권한을 검증한다.
- 역할 사용자가 `서버-상태`와 영화관 채널을 보고 반응할 수 있지만 메시지와 스레드는 작성할 수 없는지 검증한다.
- 역할 사용자가 `제어` 채널을 볼 수 없는지 검증한다.
- 참여 버튼이 역할을 한 번만 부여하고 매번 ephemeral 성공 응답을 보내는지 검증한다.
- 역할 부여 실패가 공개 메시지를 만들지 않고 ephemeral 오류를 반환하는지 검증한다.
- 소유자 공지가 정확한 채널과 Markdown 형식으로 전송되고 실제 멘션이 차단되는지 검증한다.
- 비소유자 공지와 초기화 전 공지가 거부되는지 검증한다.
- 재초기화가 저장된 역할·채널·안내 메시지를 재사용하는지 검증한다.
- Discord 리소스 생성 중간에 실패한 뒤 재시도해도 중복 리소스가 생기지 않는지 검증한다.
- 기존 공개 알림 채널을 역할 기반 권한으로 안전하게 전환하는지 검증한다.
- 전체 기존 조회, 알림, 오류·복구 및 제어 기능 회귀 테스트를 실행한다.
