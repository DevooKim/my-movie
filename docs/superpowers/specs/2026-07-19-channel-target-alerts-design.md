# 상영관 채널 예매 오픈 알림 설계

## 목표

영화별 알림 등록을 제거하고, 서비스가 지원하는 다섯 개 특별관을 각각 전용 Discord 채널에 연결한다. 사용자는 비공개 제어 채널의 버튼으로 특별관별 감시를 켜고 끄며, 새 예매 회차가 열리면 해당 특별관 채널에서 일반 Markdown 메시지를 받는다.

## Discord 구조

`/알림 초기화`는 명령을 실행한 사용자를 소유자로 기록하고 다음 비공개 구조를 자동 생성한다.

```text
영화 예매 알림
├─ 제어
├─ 메가박스-코엑스-돌비
├─ 메가박스-남현아-돌비
├─ cgv-용산-imax
├─ cgv-용산-4dx
└─ cgv-용산-screenx
```

카테고리와 채널은 `@everyone`의 보기 권한을 거부하고, 초기화한 사용자와 봇에만 보기·메시지 전송 권한을 허용한다. 초기화를 위해 봇에는 채널 관리 권한이 필요하다. 초기화한 사용자만 제어 구성요소를 조작할 수 있다.

제어 채널에는 Embed가 아닌 일반 메시지를 게시한다. 메시지는 특별관 선택 메뉴와 `알림 켜기`, `알림 끄기` 버튼을 포함하며, 다섯 특별관의 현재 상태를 `ON` 또는 `OFF`로 표시한다. 제어 메시지 ID를 저장해 상태 변경 후 같은 메시지를 갱신한다.

## 감시 대상과 채널

지원 대상은 다음 다섯 개로 고정한다.

| 대상 ID | 전용 채널 | 감시 범위 |
| --- | --- | --- |
| `megabox-coex-dolby` | `메가박스-코엑스-돌비` | 코엑스 Dolby Cinema |
| `megabox-namhyeona-dolby` | `메가박스-남현아-돌비` | 남양주현대아울렛 스페이스원 Dolby Cinema |
| `cgv-yongsan-imax` | `cgv-용산-imax` | 용산아이파크몰 IMAX |
| `cgv-yongsan-4dx` | `cgv-용산-4dx` | 용산아이파크몰 4DX |
| `cgv-yongsan-screenx` | `cgv-용산-screenx` | 용산아이파크몰 SCREENX |

한 채널은 하나의 대상만 담당하며 다른 대상의 알림을 섞지 않는다.

## 데이터 모델

사용자별 영화 구독 모델을 제거하고 단일 설치 설정과 다섯 개 대상 상태로 대체한다.

- 설치 설정: 길드 ID, 소유자 사용자 ID, 카테고리 ID, 제어 채널 ID, 제어 메시지 ID
- 대상 상태: 대상 ID, 알림 채널 ID, 활성 여부, 갱신 시각
- 기준 회차: 대상 ID와 공급자 회차 ID의 조합, 최초 확인 시각
- 폴링 기록: 공급자, 지점, 성공 여부, 조회 회차 수, 오류 요약, 시작·종료 시각

초기화 명령을 다시 실행하면 기존 설치 설정을 재사용한다. 저장된 채널이 삭제된 경우에만 누락된 채널을 다시 생성한다. 기존 영화 구독은 새 모델로 의미 있게 변환할 수 없으므로 마이그레이션 후 감시에 사용하지 않는다.

## ON/OFF 동작

대상은 초기화 직후 모두 `OFF`이다.

- `ON`: 해당 대상의 현재 전체 회차를 조회해 기준선으로 저장한 뒤 감시를 활성화한다. 이 과정에서는 알림을 보내지 않는다.
- `OFF`: 폴링 대상에서 즉시 제외한다. 기존 기준선은 정리할 수 있지만 알림 이력은 보존한다.
- `OFF → ON`: 다시 켠 시점의 현재 회차를 새 기준선으로 교체한다. 꺼져 있던 동안 열린 회차는 알리지 않는다.

ON 처리 중 공급자 조회가 실패하면 상태를 ON으로 변경하지 않고 제어 메시지에서 오류를 알려준다.

## 공급자 조회 전략

영화 카탈로그 검색과 영화별 순회를 제거하고 지점 전체 회차를 조회한다.

- CGV: 용산아이파크몰의 날짜별 전체 회차를 한 번 조회하고 특별관 코드 `02`, `03`, `04`로 4DX, IMAX, SCREENX를 분리한다.
- 메가박스: 코엑스와 남양주현대아울렛 스페이스원을 각각 영화 선택 없이 조회하고 `theabKindCd=DBC` 회차만 유지한다.

같은 지점과 날짜의 응답은 한 폴링 주기 안에서 공유한다. CGV 용산을 특별관별로 세 번 요청하지 않는다. 새 회차 판정은 공급자가 제공하는 안정적인 회차 ID를 사용한다. 좌석 수 변화만으로는 새 알림을 만들지 않는다.

## 알림 정보

알림은 Embed 없이 일반 Discord Markdown 메시지로 전송한다. 영화 제목, 날짜, 시간만 굵게 표시한다.

```text
🎬 새 예매 회차 오픈

**호프**
📅 **2026년 7월 19일**
⏰ **19:10 – 21:56**
💺 잔여 57 / 144석

CGV 용산아이파크몰 · IMAX
[앱에서 예매] [웹에서 예매]
```

같은 영화와 상영일에 여러 신규 회차가 있으면 한 메시지로 묶고 시간별 잔여석을 표시한다. 좌석 수는 알림 생성 시점의 스냅샷이며 이후 자동 수정하지 않는다.

공통으로 사용할 정보는 영화명, 지점, 상영관, 특별관/포맷, 상영일, 시작·종료 시각, 총 좌석, 잔여 좌석, 관람 등급, 예매 링크다. 값이 없는 선택 정보는 메시지에서 생략한다.

메가박스는 추가로 러닝타임, 예매 가능 여부, 포스터, 박스오피스 순위, 예매율, 무대인사·이벤트 구분을 제공한다. CGV는 상품 표시명, 자막·부가 영상 정보, 판매 종료 시각, 실제 영화 시작 시각, 포스터 경로, 회차당 최대 예매 수량을 제공한다.

## 오류 처리

- 한 지점 조회 실패는 다른 지점과 공급자의 알림을 막지 않는다.
- 실패한 폴링은 오류 요약을 저장하고 다음 주기에 재시도한다.
- 조회 실패 시 기존 기준선을 삭제하거나 갱신하지 않는다.
- Discord 채널이 삭제되거나 접근 불가능하면 해당 대상을 자동으로 OFF 처리하고 제어 채널에 상태를 반영한다.
- 동일 회차 ID는 재시작 후에도 다시 알리지 않는다.
- 공급자 응답의 좌석 필드가 누락되거나 숫자로 변환되지 않으면 좌석 표시만 생략하고 새 회차 알림은 계속 보낸다.

## 테스트 전략

- 초기화 명령이 비공개 카테고리, 제어 채널, 다섯 대상 채널을 정확히 생성하는지 검증한다.
- 소유자가 아닌 사용자의 버튼·선택 메뉴 조작을 거부하는지 검증한다.
- 대상별 ON/OFF 상태와 채널 연결이 재시작 후 유지되는지 검증한다.
- ON 및 OFF→ON 시 현재 회차를 기준선으로 만들고 기존 회차 알림을 보내지 않는지 검증한다.
- CGV 용산 응답 한 건을 IMAX·4DX·SCREENX 대상별로 분리하는지 검증한다.
- 메가박스 지점 전체 조회에서 Dolby Cinema 회차만 남기는지 검증한다.
- 신규 회차가 정확한 전용 채널에 한 번만 게시되는지 검증한다.
- 제목·날짜·시간만 굵게 표시하고 Embed를 사용하지 않는지 검증한다.
- 전체 단위 테스트, race 테스트, 정적 분석, 두 공급자의 제한된 실시간 계약 검사를 실행한다.

## 범위 밖

- 영화별 등록 및 영화 자동완성
- 일반관 알림
- 좌석 수 변화 알림
- 여러 사용자 또는 여러 길드 지원
- 알림 메시지의 실시간 좌석 수 갱신
- Embed 및 글자 색상 변경
