# 고정 특별관 알림 설계

## 목표

Discord 사용자가 임의의 극장이나 상영관을 선택하지 않고, 서비스가 지원하는 다섯 개의 `극장 + 특별관` 조합 중 하나와 영화를 선택해 예매 오픈 알림을 등록하도록 변경한다. 새 CGV 공개 JSON 조회 경로를 Go 공급자에 추가하고, 모든 공급자는 선택한 특별관의 회차만 반환한다.

## 지원 대상

지원 대상은 코드에 정적으로 정의하며 외부 카탈로그에 따라 늘어나거나 줄어들지 않는다.

| 대상 ID | 영화관 | 지점 | 지점 코드 | 특별관 | 공급자 특별관 코드 |
| --- | --- | --- | --- | --- | --- |
| `megabox-coex-dolby` | Megabox | 코엑스 | `1351` (`areaCd=10`) | Dolby Cinema | `DBC` |
| `megabox-namhyeona-dolby` | Megabox | 남양주현대아울렛 스페이스원 | `0019` (`areaCd=30`) | Dolby Cinema | `DBC` |
| `cgv-yongsan-imax` | CGV | 용산아이파크몰 | `0013` | IMAX | `tcscnsGradCd=03` |
| `cgv-yongsan-4dx` | CGV | 용산아이파크몰 | `0013` | 4DX | `tcscnsGradCd=02` |
| `cgv-yongsan-screenx` | CGV | 용산아이파크몰 | `0013` | SCREENX | `tcscnsGradCd=04` |

표시 이름은 각각 `메가박스 코엑스 · Dolby Cinema`, `메가박스 남양주현대아울렛 스페이스원 · Dolby Cinema`, `CGV 용산아이파크몰 · IMAX`, `CGV 용산아이파크몰 · 4DX`, `CGV 용산아이파크몰 · SCREENX`로 고정한다.

## 도메인 구조

`domain.AlertTarget`을 추가한다. 대상은 안정적인 `ID`, 공급자, 실제 극장 정보, 표시용 특별관 이름, 공급자가 반환하는 특별관 코드를 가진다. 다섯 개 대상의 단일 카탈로그는 별도 패키지에서 제공하며 ID 조회와 전체 목록 조회를 지원한다.

공급자 인터페이스는 임의 극장 검색을 제거하고 다음 책임을 갖는다.

- 공급자별 영화 검색
- `AlertTarget + movieID`에 대한 회차 조회
- `AlertTarget + movieID`에 대한 공식 예약 링크 생성

공급자는 대상의 공급자가 자신과 일치하는지 검증한다. Megabox는 지점 코드와 `theabKindCd=DBC`가 모두 일치하는 회차만 반환한다. CGV는 지점 코드와 `tcscnsGradCd`가 모두 일치하는 회차만 반환한다. 문자열 상영관 이름은 표시용으로만 사용하고 필터 판정에는 공급자의 구조화된 코드를 사용한다.

## Discord 명령

`/알림 등록` 옵션을 다음 두 개로 단순화한다.

1. `상영관`: 다섯 개 고정 대상을 Discord 정적 choice로 제공한다.
2. `영화`: 선택한 대상의 공급자 영화 목록을 자동완성한다.

임의 대상 ID나 다른 공급자의 영화 ID를 제출하면 DB 쓰기 전에 거부한다. 등록 완료, 목록, 삭제 자동완성, 확인 DM과 신규 회차 DM에는 고정 대상 표시 이름을 포함한다.

## 데이터 저장과 마이그레이션

구독에는 `target_id`와 `auditorium_name`을 저장한다. 중복 기준은 다음과 같다.

```text
discord_user_id + provider + target_id + movie_id
```

따라서 같은 영화에 대해 용산 IMAX, 4DX, SCREENX를 각각 등록할 수 있다. 폴링 그룹도 `provider + target_id + movie_id`로 구성하며 대상 카탈로그에서 실제 극장과 필터 코드를 해석한다.

기존 구독은 특별관을 판별할 정보가 없으므로 마이그레이션 시 `disabled`로 전환한다. 기존 전달 행은 삭제하고, 구독 고유 제약을 새 기준으로 교체하기 위해 구독 및 전달 테이블을 재구성한다. 기존 영화·극장·사용자 데이터는 보존하되 기존 구독은 다시 알림을 만들지 않는다. 사용자는 다섯 개 대상 중 하나로 다시 등록한다.

## 공급자 동작

### Megabox

현재 카탈로그와 선택 조회 계약을 유지한다. 영화 목록은 동적으로 조회하고 극장은 정적 대상의 공식 지점 코드를 사용한다. 선택 조회 응답에서 `theabKindCd=DBC`인 예약 가능한 회차만 정규화한다. `24:10`과 같은 시각은 다음 날짜 `00:10`으로 변환한다.

### CGV

공개 JSON 엔드포인트를 비로그인 HTTPS GET으로 조회한다.

- 영화 목록: `/api/v1/booking/searchAtktTopPostrList`
- 상영 가능 날짜: `/api/v1/booking/searchSiteScnscYmdListBySite`
- 회차 목록: `/api/v1/booking/searchMovScnInfo`

요청에는 회사 코드, 실제 지점 코드, 상영일을 전달한다. 쿠키, 로그인 토큰, client ID, 브라우저 자동화는 런타임 요구사항에 포함하지 않는다. 회차 응답은 영화 ID, 지점 ID, 특별관 코드가 모두 대상과 일치할 때만 사용한다. `2400` 이상 시각은 날짜를 하루씩 넘겨 `00:00` 기반 시각으로 정규화한다.

CGV 공급자는 실제 배포 컨테이너와 동일한 Go HTTP 클라이언트에서 영화 및 회차 요청이 성공하는 계약 테스트를 통과해야 레지스트리에 등록한다.

## 폴링과 알림 흐름

1. 사용자가 고정 대상과 영화를 선택한다.
2. 서비스가 대상과 영화 ID를 공급자 카탈로그로 검증한다.
3. 해당 특별관의 현재 회차만 조회해 새 구독의 기준선으로 기록한다.
4. 활성 구독을 `provider + target_id + movie_id`로 묶어 주기적으로 조회한다.
5. 새 회차를 기존 결정적 키와 비교해 중복을 제거한다.
6. 해당 대상을 구독한 사용자에게만 DM을 발송한다.

한 극장의 다른 일반관 또는 다른 특별관 회차는 기준선, DB 전달 행, 알림 메시지에 포함하지 않는다.

## 오류 처리

- 고정 대상이 공급자와 일치하지 않으면 영구 입력 오류로 처리한다.
- 공식 카탈로그에 고정 지점이나 영화가 없으면 등록을 실패시키고 초기화 구독을 정리한다.
- 공급자 응답에 필요한 식별자, 날짜, 시각 또는 특별관 코드가 없으면 계약 오류로 처리한다.
- CGV 또는 Megabox의 일시 오류는 기존 폴링 재시도 정책을 따른다.
- 기존 비활성 구독은 목록에 재등록 필요 상태로 표시하며 폴링하지 않는다.

## 테스트 전략

- 고정 대상 카탈로그가 정확히 다섯 항목이며 ID와 공급자 코드가 고유한지 검증한다.
- Discord 등록 명령이 `상영관 + 영화`만 노출하고 영화 자동완성이 대상 공급자만 호출하는지 검증한다.
- 임의 대상 ID와 다른 공급자 영화 ID를 등록 전에 거부하는지 검증한다.
- Megabox가 `DBC` 회차만 반환하고 다른 관을 제외하는지 검증한다.
- CGV 영화·날짜·회차 JSON을 파싱하고 IMAX, 4DX, SCREENX를 구조화된 코드로 구분하는지 검증한다.
- 양 공급자의 24시 이후 시각이 다음 날짜로 정규화되는지 검증한다.
- DB 마이그레이션이 기존 구독을 비활성화하고 새 중복 기준으로 같은 영화의 용산 특별관 세 종류를 모두 허용하는지 검증한다.
- 기준선 생성, 폴링, 전달 대상 매칭, 알림 표시 이름까지 통합 검증한다.
- 전체 Go 테스트, race 테스트, 정적 분석, 두 공급자 smoke test를 실행한다.

## 범위 밖

- 사용자가 임의 극장이나 특별관을 추가하는 기능
- 일반관 또는 Dolby Atmos 일반 상영관 알림
- Discord 서버 채널 알림 전환
- 좌석 수 변화 알림
- 브라우저를 서비스 런타임에 포함하는 방식
