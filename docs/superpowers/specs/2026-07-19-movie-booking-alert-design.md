# 영화 예매 오픈 알림 서비스 설계

## 1. 목표

CGV와 메가박스에서 사용자가 지정한 `영화 + 영화관 지점` 조합의 새 예매 회차를 감지하고, 등록한 Discord 사용자에게 DM으로 알린다.

초기 버전은 지정된 Discord 서버 한 곳에서만 동작한다. 사용자는 Discord 슬래시 명령과 자동완성으로 구독을 직접 관리한다. 서비스는 Bun과 TypeScript로 구현하고, Docker 컨테이너 하나와 SQLite 볼륨으로 운영한다.

## 2. 확정 요구사항

- 런타임과 패키지 관리자는 Bun을 사용한다.
- 애플리케이션 코드는 TypeScript로 작성한다.
- Discord Bot은 지정된 서버 한 곳에 길드 명령으로 등록한다.
- 알림 대상은 `영화 + 영화관 지점` 조합이다.
- 영화관, 지점, 영화는 Discord 자동완성으로 선택한다.
- 알림은 구독을 등록한 사용자에게 DM으로 전송한다.
- 구독은 사용자가 삭제할 때까지 유지한다.
- 새 날짜나 새 회차가 추가되면 계속 알린다.
- 기본 감시 주기는 5분이며 환경변수로 조절할 수 있다.
- 앱 딥링크와 웹 예매 링크를 알림에 모두 제공한다.
- SQLite 파일은 Docker 볼륨에 영속화한다.

## 3. 범위

### 포함

- CGV 및 메가박스의 영화·지점 검색
- 영화관별 예매 일정 수집과 공통 회차 형식 변환
- Discord 슬래시 명령과 자동완성
- 사용자별 구독 등록, 목록 조회, 삭제, 전체 삭제
- 최초 기준값 설정과 신규 회차 감지
- 중복 알림 방지와 DM 전송 이력
- 영화관 공식 앱 딥링크와 웹 예매 링크
- 수집 실패 및 DM 실패 기록과 제한적 재시도
- 구조화 로그와 컨테이너 상태 확인
- Docker 단일 컨테이너 배포

### 제외

- 웹 관리 화면
- 여러 Discord 서버 지원
- 일반 상영관, IMAX 등 상영관 유형 필터
- 좌석 수 또는 특정 좌석 감시
- 예매 대행 및 자동 결제
- 여러 애플리케이션 인스턴스의 동시 실행
- Redis나 외부 메시지 큐
- 관리자용 통계 대시보드

## 4. 아키텍처

초기 버전은 모듈형 단일 서비스로 구성한다. Discord Bot, 스케줄러, 영화관 연동, 알림 처리, SQLite 접근, 상태 확인 HTTP 서버를 하나의 Bun 프로세스에서 실행한다. 배포 단위는 하나지만 각 모듈은 명확한 인터페이스를 통해 통신한다.

### 모듈

- **Discord**: 슬래시 명령 등록, 자동완성, 비공개 명령 응답, DM 전송
- **Catalog**: 영화와 지점 검색, 외부 식별자 관리, 10분 메모리 캐시
- **Subscription**: 구독 등록·조회·삭제와 중복 검증
- **Provider**: 영화관별 외부 응답 수집, 검증, 공통 형식 변환, 예매 링크 생성
- **Scheduler**: 활성 구독 그룹화와 기본 5분 감시 실행
- **Notification**: 신규 회차 판별, 날짜별 메시지 묶음, 전송 이력과 재시도
- **Database**: SQLite 스키마, 마이그레이션, 트랜잭션
- **Health**: 프로세스 상태와 마지막 수집 결과를 제공하는 HTTP 엔드포인트

영화관별 코드는 다음 공통 계약을 구현한다.

```ts
interface TheaterProvider {
  searchMovies(query: string): Promise<Movie[]>;
  searchTheaters(query: string): Promise<Theater[]>;
  fetchShowtimes(input: {
    movieId: string;
    theaterId: string;
  }): Promise<Showtime[]>;
  buildBookingLinks(input: BookingLinkInput): BookingLinks;
}
```

외부 응답은 런타임 스키마로 검증한다. 응답 형식이 예상과 다르면 빈 일정으로 취급하지 않고 수집 실패로 기록하여 잘못된 삭제 판단이나 기준값 변경을 방지한다.

## 5. Discord 사용자 경험

### 명령어

- `/알림 등록 영화관:<CGV|메가박스> 지점:<자동완성> 영화:<자동완성>`
- `/알림 목록`
- `/알림 삭제 구독:<자동완성>`
- `/알림 전체삭제`
- `/알림 도움말`

명령 결과는 실행자만 볼 수 있는 비공개 응답으로 표시한다. 실제 예매 오픈 알림만 DM으로 전송한다.

등록 과정은 다음과 같다.

1. 사용자가 영화관을 선택한다.
2. 선택한 영화관의 지점명을 검색하여 자동완성 후보에서 고른다.
3. 영화명을 검색하여 자동완성 후보에서 고른다.
4. Bot이 사용자에게 DM을 보낼 수 있는지 확인한다.
5. 동일한 `사용자 + 영화관 + 지점 + 영화` 구독이 없으면 저장한다.
6. 현재 열린 회차를 기준값으로 저장한다.
7. 비공개 명령 응답과 DM으로 등록 결과를 안내한다.

DM 확인 또는 최초 일정 조회가 실패하면 구독을 활성화하지 않는다. 사용자에게 실패 원인과 재시도 방법을 비공개 응답으로 알린다. 이 정책은 등록 직후 기존 회차가 신규 회차로 오인되거나, DM을 받을 수 없는 구독이 누적되는 것을 막는다.

## 6. 감시와 신규 회차 판정

Scheduler는 활성 구독을 `영화관 + 지점 + 영화` 기준으로 묶는다. 같은 조합을 여러 사용자가 구독해도 외부 일정 조회는 주기당 한 번만 수행한다.

각 감시 실행은 다음 순서를 따른다.

1. Provider에서 현재 회차를 조회한다.
2. 응답을 공통 `Showtime` 목록으로 변환하고 검증한다.
3. 안정적인 외부 회차 ID가 있으면 이를 고유키로 사용한다.
4. 외부 회차 ID가 없으면 영화관, 지점, 영화, 상영일, 시작 시각, 상영관을 정규화해 결정적 고유키를 만든다.
5. SQLite에 없는 고유키를 신규 회차로 기록한다.
6. 해당 회차의 최초 발견보다 먼저 만들어진 활성 구독을 찾는다.
7. 아직 성공적으로 전달되지 않은 구독만 날짜별로 묶어 DM을 보낸다.
8. 전달 성공 여부와 오류를 이력에 저장한다.

일정 삭제는 사용자에게 알리지 않는다. 삭제된 회차 기록도 즉시 지우지 않아 같은 회차가 일시적으로 사라졌다가 다시 나타날 때 중복 알림이 발생하지 않게 한다.

## 7. 알림 메시지와 예매 링크

한 감시 실행에서 발견된 신규 회차는 사용자·영화·지점·상영일 기준으로 묶어 Discord Embed로 전송한다.

```text
🎬 예매가 오픈됐어요

영화       아바타 4
영화관     CGV 용산아이파크몰
상영일     2026년 12월 18일
새 회차    10:20 · 14:10 · 19:40

[앱에서 예매하기] [웹에서 예매하기]
```

- `앱에서 예매하기`는 Discord 링크 버튼이 허용하는 HTTPS 기반 공식 Universal Link 또는 App Link를 사용한다.
- `웹에서 예매하기`는 공식 웹 예매 주소를 사용한다.
- 공식 앱 링크가 영화와 지점을 직접 지정할 수 없으면 앱의 예매 진입 화면까지 연결한다.
- 공식 HTTPS 앱 링크가 제공되지 않으면 해당 영화관의 공식 앱 실행·설치 안내 링크를 앱 버튼에 사용하고, 직접 예매는 웹 버튼으로 보장한다.
- 웹 링크는 Provider가 만들 수 있는 가장 구체적인 공식 예매 화면으로 연결한다.
- 링크 생성기는 영화관별 모듈에 두며 URL 형식과 필수 식별자 포함 여부를 테스트한다.

## 8. 데이터 모델

### `subscriptions`

- 내부 구독 ID
- Discord 사용자 ID
- Provider 식별자
- 외부 지점 ID와 표시명
- 외부 영화 ID와 표시명
- 상태: `active` 또는 `disabled`
- 생성·수정 시각

`Discord 사용자 + Provider + 지점 + 영화` 조합에 고유 제약조건을 둔다.

### `showtimes`

- 결정적 회차 고유키
- Provider 식별자
- 외부 지점 ID
- 외부 영화 ID
- 외부 회차 ID가 있는 경우 해당 값
- 상영 날짜와 시작 시각
- 상영관 표시명
- 최초·최근 발견 시각

### `notification_deliveries`

- 구독 ID
- 회차 고유키
- 상태: `pending`, `sent`, `failed`
- 전송 시도 횟수
- 마지막 시도 시각과 오류 요약

`구독 ID + 회차 고유키` 조합에 고유 제약조건을 둔다.

### `poll_runs`

- Provider와 감시 조합
- 시작·종료 시각
- 성공 여부
- 조회된 회차 수
- 오류 요약

## 9. 오류 처리와 재시도

- 외부 요청 timeout은 10초로 제한한다.
- 일시적인 네트워크 오류와 서버 오류는 짧은 지수 백오프로 최대 2회 재시도한다.
- 인증 거부, 응답 검증 실패, 요청 형식 오류는 즉시 실패 처리한다.
- Provider별 동시 요청은 최대 2개로 제한하고 각 감시 주기의 작업 시작을 0~10초 범위에서 분산한다.
- 한 Provider의 실패가 다른 Provider, Discord 명령, 상태 서버를 중단시키지 않게 한다.
- 수집 실패 시 기존 회차 상태와 마지막 성공 기준값을 변경하지 않는다.
- DM 실패는 `notification_deliveries`에 기록한다.
- 일시적 DM 실패는 다음 감시 주기에 재시도하되 최초 시도를 포함해 최대 3회까지만 시도한다.
- Discord가 DM 차단을 명시적으로 반환하면 구독을 `disabled`로 바꾸고 `/알림 목록`에서 전달 불가 상태를 표시한다.

## 10. 환경설정과 운영

```env
DISCORD_BOT_TOKEN=
DISCORD_APPLICATION_ID=
DISCORD_GUILD_ID=
DATABASE_PATH=/data/my-movie.sqlite
POLL_INTERVAL_SECONDS=300
PORT=3000
TZ=Asia/Seoul
```

- Discord Bot Token만 비밀값으로 취급하고 로그에 남기지 않는다.
- Application ID와 Guild ID는 환경별 설정값으로 둔다.
- 영화관 세션 정보가 필요한 경우 코드나 고정 환경변수에 장기 저장하지 않고 Provider가 실행 중 갱신한다.
- SQLite는 WAL 모드를 사용한다.
- SQLite 파일은 `/data` Docker 볼륨에 저장한다.
- 애플리케이션은 JSON 구조화 로그를 표준 출력으로 기록한다.
- `/health`는 프로세스 실행 여부, 데이터베이스 접근 가능 여부, Provider별 마지막 성공 시각을 반환한다.
- 활성 구독이 없으면 Provider 수집 이력이 없어도 정상으로 판정한다.
- 활성 구독이 있는데 Provider의 마지막 성공 시각이 없거나 설정된 감시 주기의 두 배를 넘으면 해당 Provider를 비정상으로 표시한다.
- Docker healthcheck는 컨테이너 내부에서 `/health`를 호출한다.
- 프로세스는 `SIGTERM`을 받으면 새 감시 실행을 중지하고 진행 중인 DB 작업을 마친 뒤 종료한다.

## 11. 테스트 전략

Bun 내장 테스트 러너를 사용하고 실제 Discord나 영화관 시스템에 의존하지 않는 테스트를 기본으로 한다.

### 단위 테스트

- 영화·지점 검색 결과 정규화와 자동완성 25개 제한
- 동일 구독 중복 등록 방지
- 회차 고유키의 결정성
- 최초 등록 기준값이 알림을 만들지 않는지 검증
- 기존 회차와 신규 회차 구분
- 날짜별 회차 메시지 묶음
- 앱 딥링크와 웹 링크 생성
- 일시적 오류와 영구 오류 분류

### 통합 테스트

- SQLite 마이그레이션과 고유 제약조건
- 등록, 기준값 생성, 신규 회차 발견, DM 성공까지 전체 흐름
- 재시작 후 같은 회차를 다시 알리지 않는지 검증
- DM 실패 기록, 재시도, 차단 시 구독 비활성화
- 한 Provider 실패 중 다른 Provider 감시가 계속되는지 검증
- `/health`의 정상·비정상 상태

### 선택적 실제 연동 테스트

- 명시적으로 환경변수를 설정했을 때만 실행하는 Provider smoke test
- 영화 및 지점 검색 응답의 런타임 스키마 검증
- 특정 영화·지점 조합의 회차 조회 가능 여부
- 공식 앱 딥링크와 웹 예매 URL 형식 확인

### 배포 검증

- Docker 이미지 빌드
- 컨테이너 healthcheck 통과
- SQLite 볼륨을 유지한 컨테이너 재시작
- 재시작 전후 구독과 알림 이력 보존

## 12. 완료 기준

- 지정 Discord 서버에서 모든 `/알림` 명령이 동작한다.
- 사용자가 자동완성으로 CGV 또는 메가박스의 영화와 지점을 선택할 수 있다.
- 등록 시 현재 회차는 알리지 않고 이후 새 회차만 DM으로 받는다.
- 하나의 신규 회차를 서비스 재시작 후에도 중복 전송하지 않는다.
- 알림에 공식 앱 딥링크와 웹 예매 링크가 모두 포함된다.
- 한 영화관 연동이 실패해도 다른 영화관과 Discord 명령은 정상 동작한다.
- 기본 5분 감시 주기를 환경변수로 변경할 수 있다.
- Docker 재시작 후 구독과 알림 이력이 유지된다.
- 자동 테스트, Docker 빌드, healthcheck가 모두 통과한다.
