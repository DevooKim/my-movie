# 경계 시각 집중 조회 설계

## 목표

매 5분 경계 직후 예매가 열리는 순간을 빠르게 감지한다. CGV와 메가박스를 정각부터 30초 동안 5초 간격으로 조회하고, 각 조회에서 발견한 신규 상영 회차를 다음 조회까지 기다리지 않고 즉시 Discord 채널에 알린다.

## 실행 시간

각 집중 조회 주기는 5분 단위 경계를 기준으로 동작한다.

- 경계 10초 전: 활성 대상을 확인하고 CGV 조회 탭을 준비한다.
- 경계 시각: 첫 번째 조회를 실행한다.
- 경계 후 5초, 10초, 15초, 20초, 25초, 30초: 후속 조회를 실행한다.
- 마지막 조회 종료 후: CGV 탭을 닫고 다음 5분 경계를 기다린다.

한 주기에는 총 7번의 조회 시점이 있다. 서비스가 경계 10초 전보다 늦게 시작되면 CGV 탭을 즉시 준비하고 남아 있는 첫 조회 시각부터 참여한다. 경계 후 30초가 지난 뒤 시작되면 다음 5분 경계를 기다린다.

한 번의 조회가 다음 예정 시각까지 끝나지 않으면 같은 지점에 대한 조회를 겹쳐 실행하지 않는다. 진행 중인 조회가 끝난 직후 이미 지난 시각은 건너뛰고 아직 남은 다음 시각을 기다린다. 이는 낮은 사양의 서버에서 브라우저와 공급자 요청이 중첩되는 것을 막는다.

## 공급자별 준비와 조회

CGV 공급자는 경계 10초 전에 Lightpanda에 탭 하나를 만들고 CGV 원본 페이지를 연다. 해당 탭의 CDP 연결과 세션을 한 집중 조회 주기 동안 유지하며, 날짜와 시간표 API 요청에 같은 브라우저 문맥을 재사용한다. 용산 지점의 전체 결과는 한 번만 가져와 IMAX, 4DX, SCREENX 대상으로 분리한다.

메가박스는 브라우저를 사용하지 않으므로 별도 탭을 준비하지 않는다. 활성화된 코엑스와 남양주현대아울렛 지점을 각 조회 시점에 HTTP로 조회한다. 두 지점 및 CGV 조회는 서로 독립적으로 실행해 한 공급자나 지점의 지연이 다른 지점의 조회를 막지 않도록 한다.

CGV 준비가 실패하면 해당 시점의 CGV 조회는 일반 일회성 조회를 시도하지 않는다. 오류를 기록하고 다음 5분 경계에서 다시 준비한다. 준비된 탭이 집중 조회 도중 실패하면 남은 CGV 조회도 실패로 기록하며, 손상된 세션을 반복해서 재연결하지 않는다. 메가박스의 개별 조회 실패는 다음 예정 조회에 영향을 주지 않는다.

## 신규 회차 판정과 즉시 알림

각 조회 결과는 기존 기준 회차 ID와 즉시 비교한다. 처음 발견한 회차 ID만 전송 대기 상태로 기록하며, 이미 발견한 회차의 좌석 수 변경은 새 알림을 만들지 않는다.

각 조회가 끝날 때마다 전송 대기 알림을 바로 Discord에 보낸다. 첫 번째 조회에서 신규 회차가 발견되어도 나머지 여섯 번의 조회는 계속 실행한다. 같은 집중 조회 주기의 후속 조회에 같은 회차가 다시 포함되어도 기준 회차에 이미 기록되어 있으므로 중복 알림하지 않는다.

지점 하나의 조회가 실패해도 성공한 지점의 결과 저장과 알림 전송은 계속한다. Discord 전송 실패는 기존 전송 대기 재시도 정책을 유지한다.

## 수동 조회와 ON 처리

대상을 ON으로 바꿀 때 수행하는 기준선 조회와 내부 수동 단일 조회는 집중 조회를 사용하지 않는다. 현재 시점의 스냅샷을 한 번만 가져온다. 집중 조회는 백그라운드의 5분 단위 예약 실행에만 적용한다.

## 설정

초기 구현은 운영 요구가 고정되어 있으므로 준비 시간 10초, 조회 간격 5초, 종료 시점 30초를 별도 환경변수로 노출하지 않는다. 테스트에서는 가짜 시계와 대기 함수를 주입해 실제 시간을 기다리지 않고 실행 순서와 시각을 검증한다.

## 테스트 전략

- 첫 준비가 경계 10초 전에 실행되는지 검증한다.
- 정각부터 30초까지 5초 간격의 일곱 시점이 생성되는지 검증한다.
- 각 시점마다 CGV와 활성 메가박스 지점이 한 번씩 조회되는지 검증한다.
- 각 시점의 스냅샷 저장 직후 Discord 전송이 호출되는지 검증한다.
- 첫 조회에서 발견한 회차가 후속 조회에서 중복 전송되지 않는지 저장소 테스트로 검증한다.
- CGV의 한 탭과 CDP 세션이 일곱 조회에 재사용되고 마지막에 닫히는지 검증한다.
- 준비 실패, 개별 조회 실패, 취소 시에도 자원이 닫히고 다른 공급자의 조회가 계속되는지 검증한다.
- 오래 걸린 조회가 동일 지점에서 중첩되지 않고 이미 지난 조회 시각을 건너뛰는지 검증한다.

## 범위 밖

- 5초보다 짧은 연속 조회
- 조회 간격과 집중 조회 길이의 사용자 설정
- 좌석 수 변화 알림
- 실패한 CGV 세션의 집중 조회 도중 재생성
- 여러 집중 조회 주기의 동시 실행
