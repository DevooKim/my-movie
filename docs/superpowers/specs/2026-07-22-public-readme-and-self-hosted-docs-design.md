# 공개 README와 Self-hosted 문서 분리 설계

## 목표

저장소 첫 화면의 README를 일반 사용자가 서비스를 이해하고 이용하는 데 필요한 내용으로 재구성한다. 설치, 배포, 환경변수, 점검 및 개발 정보는 별도의 self-hosted 문서로 이동한다.

## README 구성

`README.md`는 다음 순서로 구성한다.

1. 기존 Discord 안내 이미지
2. `🏢 영화예매감시국` 제목
3. 서비스 소개
4. 이용 방법
5. 알림으로 제공되는 정보와 서버 상태 안내
6. 직접 운영하려는 사용자를 위한 self-hosted 문서 링크

README에서는 지원 영화관의 이름이나 개수, 구현 언어, 데이터베이스, Provider 요청 방식, 환경변수, 컨테이너 명령, 폴링 세부 주기 및 개발 검증 명령을 설명하지 않는다.

## Self-hosted 문서 구성

`docs/self-hosted.md`에는 기존 README의 개발 및 운영자용 내용을 다음 순서로 이동한다.

1. 서비스 구성과 동작 범위
2. Discord 애플리케이션과 Bot 권한 설정
3. 환경변수
4. Docker와 Podman 실행 방법
5. Discord 채널 초기화와 운영 명령
6. 기준선, 폴링, 오류 처리 방식
7. 앱 실행 링크 설정
8. SQLite 영속화
9. 상태 확인과 Provider 점검
10. 개발 검증 명령

기존 운영 정보는 의미가 달라지지 않도록 유지하고, README에서 self-hosted 문서로 연결한다.

## 이미지

README 상단에는 Bot 안내 메시지에서 사용하는 `internal/discordbot/assets/my-movie-pepe.png`를 상대 경로로 표시한다. 별도 복제본은 만들지 않는다.

## 검증

문서의 Markdown 구조, 상대 링크, 기존 기술 정보의 이동 여부를 정적으로 확인한다. 사용자 요청에 따라 자동 테스트는 실행하지 않는다.
