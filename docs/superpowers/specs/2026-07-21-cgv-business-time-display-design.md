# CGV 영업일 시각 표시 설계

## 목표

CGV 앱의 날짜와 시각 표기를 알림에서도 그대로 보여준다.

## 동작

CGV 응답의 상영일 `scnYmd`는 영업일로 취급한다. 시작·종료 시각이 `24:00`, `25:10`처럼 24시 이상이어도 날짜를 다음 날로 넘기거나 시간을 0시대로 바꾸지 않는다. 허용 범위는 기존과 동일한 `00:00~47:59`이며, 알림에는 `2026-08-09 24:00`처럼 표시한다.

회차 ID에는 원본 영업일이 이미 포함되므로 ID와 중복 알림 판정은 변경하지 않는다.

동일 회차 ID가 DB에 이미 달력 시각으로 저장되어 있으면 다음 조회의 upsert에서 `play_date`, `starts_at`, `ends_at`을 최신 CGV 표기로 갱신한다. 기존 pending 알림도 교정된 값을 사용한다.

## 검증

`2400`, `2510`, 일반 시각을 각각 변환해 영업일 날짜와 원본 시간 표현이 유지되는지 검사한다. Provider 결과의 종료 시각 `2410`도 `24:10`으로 유지되는지 확인한다. 같은 회차를 교정된 날짜·시간으로 다시 저장했을 때 pending delivery가 최신 표기를 읽는지도 검증한다.
