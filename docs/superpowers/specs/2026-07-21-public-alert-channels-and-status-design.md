# 공개 알림 채널과 서버 상태 채널 설계

## 목표

영화관별 예매 오픈 알림을 서버 구성원과 함께 볼 수 있도록 알림 채널을 공개한다. 제어 기능과 상세 오류는 설치 소유자에게만 공개하고, 일반 구성원에게는 안전하게 요약한 서비스 상태만 제공한다.

## 채널 구조

`/알림 초기화`는 다음 구조를 생성하거나 기존 구조를 갱신한다.

```text
영화 예매 알림
├─ 제어
├─ 서버-상태
├─ 메가박스-코엑스-돌비
├─ 메가박스-남현아-돌비
├─ cgv-용산-imax
├─ cgv-용산-4dx
└─ cgv-용산-screenx
```

`제어` 채널은 기존처럼 초기화한 소유자와 봇만 접근할 수 있다. `서버-상태`와 영화관별 알림 채널은 모든 서버 구성원이 볼 수 있다.

## 권한

공개 채널에서 `@everyone`은 채널 보기, 메시지 기록 읽기, 반응 추가를 할 수 있다. 메시지와 스레드 작성은 허용하지 않는다. 봇은 채널 보기, 기록 읽기, 메시지 작성 및 반응에 필요한 권한을 가진다. Discord의 관리자 권한을 가진 구성원은 플랫폼의 관리자 권한에 따라 채널을 관리하고 메시지를 작성할 수 있다.

카테고리는 공개 읽기 전용 권한을 기본으로 사용한다. 영화관별 알림 채널과 `서버-상태` 채널은 이 권한을 적용한다. `제어` 채널에는 별도 비공개 권한을 적용하여 카테고리의 공개 범위를 덮어쓴다.

기존 설치에서 `/알림 초기화`를 다시 실행하면 기존 채널 ID를 유지하면서 새 권한을 적용하고 `서버-상태` 채널이 없으면 생성한다.

## 오류 및 복구 알림

기존 비공개 `제어` 채널 알림은 변경하지 않는다. 첫 조회 실패 시 공급자, 지점, 상세 오류 내용을 전송하고, 같은 지점의 연속 실패는 중복 전송하지 않는다. 실패 상태에서 처음 정상 응답을 받으면 기존 정상화 메시지를 전송한다.

공개 `서버-상태` 채널에는 같은 상태 전이를 다음과 같이 요약해서 함께 전송한다.

```text
⚠️ CGV · 용산아이파크몰 조회가 원활하지 않습니다
✅ CGV · 용산아이파크몰 조회가 정상화되었습니다
```

공개 메시지에는 HTTP 상태 코드, 네트워크 오류, 예외, 내부 구현 이름 및 원문 오류를 포함하지 않는다. 상세 오류는 비공개 `제어` 채널과 서버 로그에만 남긴다.

공개 상태 알림은 기존 `poll_alert_states`의 정상·실패 전이를 공유한다. 정상 상태에서 실패로 바뀔 때 실패 메시지를 한 번 보내고, 실패 상태에서 정상으로 바뀔 때 복구 메시지를 한 번 보낸다. 알림 전송 중 일부 채널만 실패하면 성공한 채널에 같은 상태 메시지를 중복 전송하지 않도록 채널별 전송 결과를 다룬다.

## 데이터 저장

`installations`에 `status_channel_id`를 추가하여 길드의 공개 상태 채널을 저장한다. 기존 데이터베이스는 마이그레이션 시 빈 값으로 시작하며, 다음 `/알림 초기화`에서 채널을 생성하고 ID를 저장한다.

`poll_alert_states`에는 제어 채널과 상태 채널의 전달 완료 여부를 각각 저장한다. 한 채널만 전송에 실패하면 성공한 채널은 중복 전송하지 않고 실패한 채널만 다음 동일 상태 조회에서 재시도한다. 장애 메시지가 전달되지 않은 채널에는 해당 장애의 복구 메시지를 보내지 않는다.

영화관별 채널 ID, 제어 채널 ID, 소유자 정보와 조회 실패 상태는 기존 데이터를 유지한다.

## 오류 처리

- 공개 상태 채널을 생성하거나 권한을 갱신하지 못하면 `/알림 초기화`를 실패 처리하고 원인을 명령 응답에 표시한다.
- 상세 제어 알림 전송 실패와 공개 상태 알림 전송 실패는 서버 로그에 남긴다.
- Discord 알림 전송 실패가 영화관 조회 결과와 기준선을 손상시키지 않도록 한다.
- 초기화 전이거나 상태 채널 ID가 없는 기존 설치에서는 상세 제어 알림만 유지한다. 공개 상태 알림은 초기화를 다시 실행한 뒤부터 전송한다.

## 검증

- 신규 초기화가 `제어`, `서버-상태`, 다섯 영화관 채널을 생성하는지 검증한다.
- 기존 초기화를 다시 실행했을 때 채널을 중복 생성하지 않고 권한을 갱신하는지 검증한다.
- 일반 구성원이 공개 채널을 보고 기록을 읽고 반응을 추가할 수 있지만 메시지를 작성할 수 없는 권한 덮어쓰기를 검증한다.
- 일반 구성원이 `제어` 채널을 볼 수 없는지 검증한다.
- 첫 실패 시 제어 채널에는 상세 오류, 상태 채널에는 요약 오류가 각각 한 번 전송되는지 검증한다.
- 연속 실패가 두 채널 모두에서 중복 알림을 만들지 않는지 검증한다.
- 복구 시 두 채널에 각 목적에 맞는 정상화 메시지가 한 번 전송되는지 검증한다.
- 상태 채널이 아직 없는 기존 설치에서도 상세 제어 알림이 계속 동작하는지 검증한다.
