# 조회 오류 알림 설계

## 목표

CGV와 메가박스의 지점 조회가 실패하면 Discord `제어` 채널에 오류를 알리고, 같은 지점이 다시 정상 응답하면 복구 알림을 보낸다. 반복 실패 중에는 같은 오류를 다시 보내지 않는다.

## 알림 흐름

각 공급자·지점 조합의 마지막 오류 상태를 SQLite에 보존한다.

- 정상 상태에서 조회 실패: 오류 알림을 한 번 전송하고 상태를 오류로 저장한다.
- 오류 상태에서 다시 조회 실패: 알림을 보내지 않고 오류 상태만 유지한다.
- 오류 상태에서 조회 성공: 복구 알림을 한 번 전송하고 상태를 정상으로 저장한다.
- 정상 상태에서 조회 성공: 알림을 보내지 않는다.

오류 알림 메시지는 공급자와 지점 이름, 발생 시각, 간결한 오류 요약을 포함한다. 복구 메시지는 공급자와 지점 이름, 정상화 시각을 포함한다. 전용 상영관 채널에는 예매 오픈 알림만 계속 게시한다.

## 범위와 오류 판정

공급자에서 시간표 스냅샷을 가져오는 요청이 실패한 경우만 조회 오류로 처리한다. 예매 알림 전달, SQLite 기록, 상태 갱신 오류는 별도 서비스 오류로 두며 조회 오류 알림을 만들지 않는다.

CGV 사전 탭 준비 실패와 준비된 탭의 조회 실패도 CGV 지점 조회 오류로 처리한다. 메가박스의 지점별 HTTP 요청 실패도 해당 지점의 조회 오류로 처리한다. 한 지점 오류는 다른 지점의 조회와 알림을 막지 않는다.

## 저장과 전달

오류 상태에는 공급자, 지점 ID, 오류 여부, 마지막 오류 요약, 갱신 시각을 저장한다. 상태 변경이 필요한 경우 Discord 메시지를 먼저 전송하고, 전송 성공 후에만 상태를 갱신한다. Discord 전송이 실패하면 다음 조회에서 같은 상태 변경을 다시 시도한다.

초기화 전에는 제어 채널이 없으므로 오류 알림을 보내지 않는다. 초기화 후에는 저장된 제어 채널을 사용한다. 제어 채널이 접근 불가능하면 오류를 반환해 서버 로그에 남기지만, 예매 오픈 알림 처리에는 영향을 주지 않는다.

## 테스트 전략

- 첫 조회 실패가 제어 채널에 오류 메시지 한 번을 보내는지 검증한다.
- 연속 실패가 추가 오류 메시지를 만들지 않는지 검증한다.
- 오류 뒤 첫 성공이 복구 메시지 한 번을 보내는지 검증한다.
- 정상 상태의 성공이 메시지를 만들지 않는지 검증한다.
- 상태가 저장소 재생성 후에도 유지되는지 검증한다.
- Discord 전송 실패가 상태를 바꾸지 않아 다음 조회에서 재시도되는지 검증한다.
- CGV 준비 실패 및 메가박스 지점 실패가 다른 지점의 집중 조회·예매 알림을 막지 않는지 검증한다.

## 범위 밖

- 전용 상영관 채널에 기술 오류 메시지 전송
- 오류별 사용자 설정, 수신 시간 설정, 외부 모니터링 연동
- 예매 알림 전달 실패를 조회 오류로 변환
